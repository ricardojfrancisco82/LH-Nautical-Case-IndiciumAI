1 Um passo a frente, com sistemas de recomendação e previsões de demanda

O mundo contemporâneo ive uma inflação de dados, e a LH Náutica não é exceção. Vemos até aqui algumas dificuldades na administração e tratamento cotidiano desses dados, o que tem gerado insights muitas vezes confusos, e obrigando os gestores a trabalharem com o “feeeling” Entretanto, se pudessemos fazer algo além de apenas arrumar os dados e dar um fluxo dinâmico organizado para esta empresa. Para tal, verificamos o rpoblema de gerenciamento de estoque e sugerimos um modelo preditivo que diga exatamente quantas unidades venderemos no próximo mês para ajustar as compras com fornecedores.

1.1 Parâmetros utilizados no ensaio

  • O período de treino foram os dados até 31/12/2023.
  • O período de teste do modelo foi todo o mês de Janeiro de 2024.
  • A previsão foi feita em base diária.
  • Não foram utilizados dados futuros no treino.
  • Consideramos apenas o produto: “Motor de Popa Yamaha Evo Dash 155HP”, para fins de estudo de caso

1.2 Teste 1 - usando um sistema de series de tempo com Prophet

Primeiramente usamos a ferramenta Prophet, criada pelo Facebook para previsão de séries temporais, especialmente útil em dados com forte sazonalidade e tendências não lineares. Ele é simples de usar, robusto contra valores ausentes e outliers. Sua sintaxe é simples, não necessitando de muita expertise para lidar com o modelo, sabe lidar bem com dados faltantes e é justamente o tipo de ferramente em que negócios que não tem tanto orçamento para BigData podem recorrer.

1.4 Teste 3 - Um simples sistema de recomendação

Sistemas de recomendação são ferramentas corriqueiras de e-commerce, porém, dependem de uma grande numeros de dados que se correlacionem entre si. Aqui fizemos um pequeno ensaio para veridficar se, dentro do escopo de nossos dados, podemos geral algo de semelhante para auxiliar os vendedores, ou até mesmo poder instruir melhor nosso site de e-commerce. Aqui, criamos um cenário tentando vincular quais itens poderiamos recomendar quando o cliente se interessa pelo item “GPS Garmin Vortex Maré Drift”. Para tal, usamos uma ferramenta matemártica, a similaridade de cosseno é o resultado da relação existente entre as vendas de produtos, com a conversão das ocorrências como um vetor, onde um ângulo de zero representa que os vetores apontam para a mesma direção, ou seja, são 100%$ compatíveis entre si. Para construir um sistema de recomendação de produtos, com base no produto anterior selecionado e por similaridades, o primeiro passo é criar uma matriz que relaciona as aquisições dos clientes e os produtos em si. O resultado dessa matriz, é no caso uma linha que registra a quantidade de ocorrências de compra dos produtos, que situam-se nas colunas. *O próximo passo foi converter os valores registrados em booleano, pois só queríamos saber da convergência (haver compras relacionados) ou não dos produtos, chegando a similaridade de cosseno a ser comaprada entre os itens.
